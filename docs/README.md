# Coach

Проект Coach: Telegram-бот + Mini App для учета тренировок, симптомов, анализов, приемов и документов.

## Цель v1
- принимать текст, PDF, DOCX, фото и голосовые сообщения в Telegram;
- автоматически раскладывать данные по сущностям: тренировки, симптомы, анализы, приемы, документы, протоколы;
- показывать историю и динамику в Mini App;
- хранить фронт отдельно от боевого бэкенда.

## Базовая архитектура
- GitHub repo `coach` — исходники проекта
- GitHub Pages / отдельный фронт — Mini App UI
- VDSina `bots/coach` — бот, API, БД, обработка файлов
- Docker-only деплой на VDSina

## Слои системы
1. `bot` — Telegram Bot webhook/polling layer
2. `api` — backend API для Mini App и ingestion
3. `worker` — обработка PDF/DOCX/картинок/очереди задач
4. `db` — PostgreSQL
5. `storage` — локальное файловое хранилище на VDSina

## Первые практические этапы
1. Создать отдельный git-репозиторий `coach`
2. Перенести/собрать Mini App фронт в проект
3. Описать JSON/DB-модель сущностей
4. Поднять Telegram bot ingest для текста и документов
5. Подключить storage + базу + первичный парсинг
6. Завести деплой в `bots/coach` на VDSina
